Brief overview of the core concepts of SemMap SemMap makes use of the following concepts:
  • Concept: This is a (graph-pattern, variable) pair, such as ({ ?s a Commitment}, ?s). It describes an initial set of resources
  • Focus path: Based on the concept, a focus path leads from the concept-variable to a different set of variables, for which information should be displayed. Usually it is empty, which means that the focus resources are the ones identified by the concept.
  • Geo-path: The path from the concept to the geo-resources. As such we can add geo-constraints (e.g. bounding box) on it.
  • Navigation path: The path the user navigated along by following the properties of the resources. The facets presented to the user are those that apply to the current set of resources the user navigated to. The count of each facet shows, how many corresponding resources there are on the focus path.
    • This means: If the concept is ({ ?s a Commitment}, ?s), the focus path empty, and the navigation path is "beneficiary" then:
      The user sees the facets of beneficiaries, but the facet counts show, how many commitments there are.
